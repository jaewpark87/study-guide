## Part 1: 파일 내용 보기 📄

### 실습 파일 만들기

먼저 연습용 파일을 만들어 봅시다:

```bash
cd ~/Desktop
mkdir shell_practice && cd shell_practice

echo "1. 사과" > fruits.txt
echo "2. 바나나" >> fruits.txt
echo "3. 포도" >> fruits.txt
echo "4. 딸기" >> fruits.txt
echo "5. 망고" >> fruits.txt
```

> 💡 **컴퓨터 공학 개념 — 리다이렉션 (Redirection)**
>
> `>` 와 `>>` 는 명령어의 출력을 **화면 대신 파일로 보내는** 기능입니다.
>
> | 기호 | 동작 | 비유 |
> | --- | --- | --- |
> | `>` | 덮어쓰기 (새로 만듦) | 새 노트에 쓰기 |
> | `>>` | 추가하기 (뒤에 이어서) | 기존 노트에 덧붙이기 |
>
> ```bash
> echo "hello" > file.txt    # file.txt = "hello"
> echo "world" > file.txt    # file.txt = "world" (덮어씀!)
> echo "!!!" >> file.txt     # file.txt = "world\n!!!" (추가!)
> ```

### `cat` — 파일 전체 보기

```bash
cat fruits.txt
```
```
1. 사과
2. 바나나
3. 포도
4. 딸기
5. 망고
```

> `cat`은 "Concatenate(이어 붙이기)"의 줄임말이에요.

### `head` / `tail` — 앞/뒤만 보기

```bash
head -2 fruits.txt    # 처음 2줄만
# → 1. 사과
# → 2. 바나나

tail -2 fruits.txt    # 마지막 2줄만
# → 4. 딸기
# → 5. 망고
```

> 비유: `head` = 책의 **첫 페이지**, `tail` = 책의 **마지막 페이지**

### `less` — 긴 파일 편하게 읽기

```bash
less fruits.txt
```

| 키 | 동작 |
| --- | --- |
| `Space` | 다음 페이지 |
| `b` | 이전 페이지 |
| `/검색어` | 검색 |
| `q` | 종료 |

> `less`는 **전자책 리더기** 📖 같아요! 긴 파일을 스크롤하면서 볼 수 있습니다.

### `wc` — 파일 통계 (Word Count)

```bash
wc fruits.txt
#  5  10  56 fruits.txt
#  ↑   ↑   ↑
#  줄  단어  바이트
```

> 💡 **컴퓨터 공학 개념 — 표준 입출력 (Standard I/O)**
>
> 모든 프로그램에는 3가지 **데이터 통로**가 있습니다:
>
> | 이름 | 번호 | 역할 | 비유 |
> | --- | --- | --- | --- |
> | **stdin** | 0 | 입력 (키보드) | 귀 👂 |
> | **stdout** | 1 | 출력 (화면) | 입 👄 |
> | **stderr** | 2 | 에러 출력 | 빨간 경고등 🚨 |
>
> `>` 리다이렉션은 stdout(화면에 나올 내용)을 **파일로 우회**시키는 것입니다!

## Part 2: 검색하기 🔍

### `grep` — 파일 내용에서 찾기

```bash
# "포도" 포함된 줄 찾기
grep "포도" fruits.txt
# → 3. 포도

# 여러 파일에서 찾기
grep "error" *.log

# 대소문자 무시 (-i)
grep -i "apple" fruits.txt

# 하위 폴더까지 검색 (-r)
grep -r "TODO" my_project/

# 줄 번호도 보기 (-n)
grep -n "딸기" fruits.txt
# → 4:4. 딸기
```

> `grep`은 **돋보기** 🔍 입니다! 파일 속에서 원하는 단어를 찾아줘요.

### `find` — 파일 이름으로 찾기

```bash
# 현재 폴더에서 .txt 파일 찾기
find . -name "*.txt"

# 홈 폴더에서 "hello"가 이름에 포함된 파일
find ~ -name "*hello*"
```

> 💡 **`grep` vs `find` 차이점:**
>
> | | `grep` | `find` |
> | --- | --- | --- |
> | 뭘 검색? | 파일 **안의 내용** | **파일 이름** |
> | 비유 | 책 내용 검색 📖 | 도서관에서 책 찾기 🏛️ |
> | 예시 | `grep "에러" log.txt` | `find . -name "*.log"` |

> 💡 **컴퓨터 공학 개념 — 와일드카드 (Wildcard)**
>
> `*`는 **"아무 글자나 다"**를 뜻하는 **와일드카드**입니다.
>
> | 패턴 | 의미 | 매칭 예시 |
> | --- | --- | --- |
> | `*.txt` | .txt로 끝나는 모든 파일 | hello.txt, note.txt |
> | `test_*` | test_로 시작하는 모든 것 | test_1, test_abc |
> | `*.py` | 파이썬 파일 전부 | app.py, main.py |
>
> 카드 게임에서 와일드카드가 "아무 카드 대신 쓸 수 있는" 것과 같아요! 🃏

## Part 3: 파이프 — 명령어 연결하기

### 파이프 `|` 란?

파이프(`|`)는 한 명령어의 **출력**을 다음 명령어의 **입력**으로 전달합니다:

```bash
# ls 결과에서 ".txt" 파일만 찾기
ls | grep ".txt"

# 파일 줄 수 세기
cat fruits.txt | wc -l
# → 5

# 히스토리에서 "git" 관련 명령어 찾기
history | grep "git"
```

> 💡 **컴퓨터 공학 개념 — 파이프 (Pipe)**
>
> 파이프는 **UNIX 철학**의 핵심입니다:
>
> > "한 가지 일을 잘 하는 작은 프로그램을 만들고, 서로 연결하라"
>
> ```
> 명령어1  |  명령어2  |  명령어3
>  (출력) → (입력→출력) → (입력)
> ```
>
> 비유: 공장의 **컨베이어 벨트** 🏭
> ```
> [ls]  →  결과물  →  [grep ".txt"]  →  필터된 결과  →  [wc -l]  →  개수
> ```
>
> 각 기계(명령어)는 한 가지만 잘 하고, 벨트(파이프)로 연결하면 복잡한 작업도 가능!

### 실전 예시

```bash
# 가장 큰 파일 3개 찾기
ls -lS | head -4

# 폴더만 보기
ls -l | grep "^d"

# 현재 실행 중인 프로세스 중 python 찾기
ps aux | grep python
```

## Part 4: 터미널 단축키 & 꿀팁 ⚡

### 키보드 단축키

| 단축키 | 동작 |
| --- | --- |
| `Tab` | **자동 완성** (가장 유용!⭐) |
| `↑` / `↓` | 이전/다음 명령어 히스토리 |
| `Ctrl + C` | 실행 중인 명령어 **중단** |
| `Ctrl + L` | 화면 **지우기** |
| `Ctrl + A` | 줄 **맨 앞**으로 |
| `Ctrl + E` | 줄 **맨 뒤**로 |
| `Ctrl + R` | 명령어 **검색** |

> 💡 `Tab`은 **터미널의 마법 키**입니다!
> `cd Desk` + `Tab` → 자동으로 `cd Desktop/` 완성!
> 두 번 탭하면 후보를 모두 보여줍니다.

### 맥 전용 꿀 명령어

| 명령어 | 기능 |
| --- | --- |
| `open .` | 현재 폴더를 **Finder에서 열기** |
| `open file.pdf` | 파일을 기본 앱으로 열기 |
| `which python` | 프로그램 위치 찾기 |
| `man ls` | 명령어 설명서 (`q`로 종료) |
| `history` | 이전에 쓴 명령어 보기 |

> 💡 **컴퓨터 공학 개념 — 환경 변수 (Environment Variable)**
>
> 터미널에는 **환경 변수**라는 "시스템 설정값"이 있습니다:
>
> ```bash
> echo $HOME     # 홈 폴더 경로
> echo $USER     # 현재 사용자 이름
> echo $PATH     # 프로그램 검색 경로
> ```
>
> `$PATH`는 특히 중요합니다! 명령어를 입력하면 Shell은 `$PATH`에 등록된 폴더들을 순서대로 뒤져서 프로그램을 찾습니다.
>
> `python`을 입력하면 → `$PATH`의 폴더들에서 `python` 파일을 찾음 → 실행!

### 🛠️ Cheat Sheet

```
📄 파일 보기
  cat 파일      전체 보기
  head -n 파일  처음 n줄
  tail -n 파일  마지막 n줄
  less 파일     스크롤 보기 (q로 종료)
  wc -l 파일    줄 수 세기

🔍 검색
  grep 텍스트 파일     내용 검색
  grep -r 텍스트 폴더  하위폴더까지 검색
  find . -name "패턴"  파일 이름 검색

🔗 리다이렉션 & 파이프
  명령어 > 파일     결과 저장 (덮어쓰기)
  명령어 >> 파일    결과 추가
  명령어1 | 명령어2  연결 (파이프)

⚡ 단축키
  Tab          자동완성
  ↑↓           히스토리
  Ctrl+C       중단
  Ctrl+L       화면 지우기
  Ctrl+R       명령어 검색
```
